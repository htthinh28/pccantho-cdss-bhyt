1. TÓM TẮT ĐIỀU HÀNH
================================================================================

Module báo cáo là lớp “đọc kho — tổng hợp — hiển thị — xuất” cho ba nhánh nghiệp vụ:
Quản trị (M5/M6), Chuyên môn (M7), Doanh thu BHYT (M8). Luồng lõi không ghi
ngược metadata vào kho; mọi số liệu là snapshot tại thời điểm gọi
taiNguonVaMoHinhMuc5.

Tài liệu 2.0.3: (i) chiến lược dashboard y tế / trực quan hoá 2025–2026 (outcome-first, WCAG, Agentic AI lộ trình);
  (ii) drill CM-00 trong Hub + drill_registry khai báo (SPEC-VIZ-2.0.3);
  (iii) cache fingerprint + boQuaCache; (iv) Gifted + Victory Skia khi cần; (v) Excel / PDF / ZIP CSV / JSON snapshot-3;
  (vi) backlog: FHIR, BigQuery, skeleton/debounce — mục 5–7 và 10.

2. XU HƯỚNG UI/UX & TRỰC QUAN HÓA (KHẢO SÁT — ÁP DỤNG CHO MODULE)
================================================================================

2.1 Trích yếu phục vụ quản trị & chất lượng y tế
  - Ưu tiên tín hiệu trong vài giây đầu: KPI rủi ro, top khoa/rule, màu phân cấp.
  - Drill-down có lộ trình: tổng → theo khoa / nhóm lỗi → danh sách hồ sơ / dòng lỗi.
  - Progressive disclosure: bảng tóm tắt trước, chi tiết mở theo nhu cầu; breadcrumb.
  - Giảm chuyển ngữ cảnh: gắn số liệu với rule, namespace, tab quản trị gợi ý.
  - Dashboard “chủ động”: gợi ý hành động (đã có phần văn bản trong CM-04); mở rộng
    UI nút lọc nhanh theo metadata (đặc tả dữ liệu 2.0 — khối hien_thi_bao_cao).
  - Thời gian thực / gần thời gian thực: dấu thời gian dữ liệu + làm mới có chu kỳ;
    chuẩn bị sẵn hook cho push sau này (SSE/WebSocket) nếu backend có.

2.2 Trực quan hóa dữ liệu hiện đại
  - Biểu đồ tương tác (tooltip, chọn điểm, lọc cử chỉ), heatmap cho ma trận khoa×nhóm.
  - So sánh chuỗi thời gian (M8 theo tháng) dạng line/area khi dữ liệu đủ mịn.
  - Tránh “mọi thứ cùng trọng số thị giác”: dùng theme và ngưỡng cảnh báo (đặc tả VIZ).

2.3 Nguồn đọc thêm (không ràng buộc pháp lý BHYT)

4. THÀNH PHẦN GIAO DIỆN VÀ QUY TRÌNH XUẤT BÁO CÁO
================================================================================

4.1 Thành phần UI hiện có
  - baocao_va_thongke.jsx → re-export BaoCaoHub (ổn định deep link / RBAC).
  - BaoCaoHub.jsx: chọn nhánh QUAN_TRI | CHUYEN_MON | DOANH_THU_BHYT; tab M5/M6;
    định nghĩa cột bảng theo từng dataset (gồm BC-DT-01 Top rule: ma_rule, ten_quy_tac, so_loi, Σ rủi ro);
    nút xuất Excel / PDF / ZIP CSV / JSON snapshot; header/trình bày hub gọn theo design hiện tại.
  - BangMoHinhMuc5.jsx: lưới cuộn ngang, font body tối thiểu theo chỉnh gần đây.
  - BieuDoThanhNgang.jsx: thanh ngang Animated (RN) — vẫn dùng nơi phù hợp tương thích cũ.
  - HeatmapCm00.jsx: ma trận khoa × nhóm vi phạm (CM-00), sự kiện onChonO kích hoạt drill.
  - BieuDoGiftedBarNgang.jsx / BieuDoGiftedCotDoc.jsx: biểu đồ thanh ngang / cột dọc (react-native-gifted-charts,
    cần expo-linear-gradient). M7/M8 dùng các component này cho nhiều widget.
  - bao_cao_drill_chuan.js: chuanHoaMaKhoaBaoCao, khoaDrillKhop — đồng bộ mã khoa khi lọc drill.
  - FlatList: danh sách dài (kể drill hồ sơ); không dùng @shopify/flash-list trên bundle web (tránh lỗi resolve).

4.2 Quy trình xuất (chi tiết)
  (1) Người dùng chọn nhánh và (nếu Quản trị) M5 hoặc M6.
  (2) Hub gọi layCacBangDeXuat({ nhanh, quanTriThe, tai }) với tai.moHinh | muc6|7|8.
  (3) Manifest thêm từng sheet: cột suy ra từ keys hàng đầu (hoặc cột cố định M5 — ví dụ FACT_CANH_BAO có ten_quy_tac).
  (4) Sheet rỗng → một dòng placeholder.
  (5) _META ghi tieu_de, nhanh, quan_tri_the, so_ho_so, thoi_diem_xuat (ISO).
  (6) xuatExcelBaoCao: json_to_sheet theo label; tên sheet ≤31 ký tự, trùng tên → hậu tố.
  (7) PDF: render HTML từ cùng sheets (expo-print).
  (8) Tuỳ chọn Hub: xuatZipCsvBaoCao (ZIP nhiều CSV UTF-8), xuatJsonSnapshotBaoCao (snapshot JSON cho BI / meta).

6. THƯ VIỆN BIỂU ĐỒ & TỐI ƯU LUỒNG TƯƠNG TÁC (REACT NATIVE / EXPO)
================================================================================

6.1 Lựa chọn gợi ý (theo độ ưu tiên kỹ thuật)

  (A) victory-native (+ @shopify/react-native-skia, react-native-reanimated,
      react-native-gesture-handler)
      - Ưu: GPU Skia, tooltip/chartPress, hiệu năng tốt tập dữ liệu lớn.
      - Nhược: cần Expo development build / EAS — không chạy đầy đủ trên Expo Go.
      - Tài liệu: https://nearform.com/open-source/victory-native/docs/getting-started
      - Repo: https://github.com/FormidableLabs/victory-native-xl

  (B) react-native-gifted-charts (+ expo-linear-gradient) — ĐANG DÙNG trong BaoCaoHub cho bar/column.
      - Ưu: cài đặt nhẹ, chạy Expo Go / web bundle hiện tại; gradient hỗ trợ style chart.
      - Nhược: SVG/JS — hạn chế hơn khi N > vài nghìn điểm; theo dõi kích thước bundle.

  (C) Web-only (nếu bundle web tách): @nivo/* hoặc Recharts trong màn web dashboard
      — chỉ khi dự án chấp nhận song song code path Platform.OS === 'web'.

6.2 Tối ưu luồng tương tác người dùng
  - Giảm số lần taiNguonVaMoHinhMuc5: cache trong bao_cao_service (fingerprint nguồn);
    boQuaCache=true để làm mới cứng.
  - Virtualized list: FlatList cho danh sách dài (đồng nhất native/web; tránh FlashList trên web).
  - Skeleton / ActivityIndicator trong lần tải đầu; debounce chuyển tab nhánh.
  - Haptics nhẹ khi chuyển nhánh (đã có pattern) + tránh spam khi polling.

6.3 Babel / native
  - Reanimated: plugin phải là mục cuối trong babel.config.js khi dùng Victory Native.
